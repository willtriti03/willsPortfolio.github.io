해커가 육안으로 프로그램 메모리를 확인할 수 있다!

제작 동기

대학교내 BottomUp 창의력 경진대회에 출품하기 위해 VR게임으로 기획한 게임입니다.
코딩 수업이 초중고학생들에게 의무화 되면서 코딩을 게임에 녹여내여 재밌게 접근시키고 싶었습니다. 그래서 몸으로 직접 움직이면서 할 수 있는 VR게임을 선택하였습니다.

게임에 나오는 내용들은 SWAP알고리즘등 간단한 알고리즘을 담았습니다.


게임 장르

VR 3D 1인칭 모험 & 퍼즐


게임 개요

주어진 코드가 바르게 동작하기 위해 메모리에 접근하여 변수의 값을 수정하라


게임 요소

다양한 코딩 요소
- 올바른 IF문으로 바꾸기, SWAP 알고리즘등이 문제로 주어진다.
- 플레이어는 변수 값을 조절하거나 값 데이터를 찾아 변수자리로 옮겨줘야한다.
- 코드를 해석해야 문제 해결이 가능하다.

한정된 시야
- 시야는 1인칭으로 보기 때문에 보이는 요소들이 적다.
- 모든 벽들은 각각의 메모리이고 마우스에 닿아 있는 타겟 블럭만 서서히 메모리 값이 나타난다.
- 한정된 시야를 해결하기 위해 머리 위에 미니맵이 떠있다.

문제 해결을 방해하는 적을 회피
- 플레이어는 불필요한 메모리로 취급하기 때문에 적 캐릭터의 이름은
불필요 메모리를 정리해주는 Garbage Collector이다.
- Garbage Collector는 플레이어와 닿으면 게임을 끝내는 요소이다.
- Garbage Collector는 특정 지역을 순찰한다.

코드의 해석
- 모든 특정 작동 함수는 직관적인 이름으로 적혀있다.
- 미니맵의 정보와 조합하여 어느 문을 열고 닫을지를 결정해야한다.
- Garbage Collector가 갇혀있는 문을 열지 않게 조심한다.


게임 승리 / 패배 요소

승리 요소
- 모든 코드를 풀고 메모리를 탈출한다.

패배 요소
- Garbage Collector에게 Garbage Collecting 당한다.

회고

이 프로젝트에서는 Unity를 능숙하게 다루는 형과 작업하였는데 기술적인 부분을 잘 생각하여 개발해 주셨다.
팀은 처음에는 3인이었지만 한 명이 거의 참여하지 못하면서 사실상 기획과 개발 2인 프로젝트였다.VR이 3D이기때문에 3D모델러를 구하는건 상당히 어려워 디자인면으로는 상당히 아쉬웠다.
두 번째 모바일 VR게임 프로젝트였지만 여전히 VR은 어려운 것 같다.
스마트폰으로 VR게임을 제작 시 특정 이상의 콘텐츠와 오브젝트를 넣으면 프레임이 잘 안 나오기 때문에 콘텐츠와 오브젝트의 조절이 중요했습니다.
불필요한 기능들은 모두 조절하였지만 그래도 조금은 프레임이 부족한 모습을 보여 아쉬웠다.
